**WebApplicationContext** — это центральный интерфейс в Spring Framework, который расширяет базовый ApplicationContext и добавляет специфичные для веб-приложений возможности. Он предоставляет дополнительные функции, необходимые для работы с веб-приложениями, такими как веб-приложения на основе сервлетов.

Основные особенности **WebApplicationContext**:

- Расширение ApplicationContext:
WebApplicationContext расширяет ApplicationContext, предоставляя все его возможности, такие как управление бинами, внедрение зависимостей и управление конфигурацией.

-  Интеграция с сервлетами:
WebApplicationContext интегрируется с сервлет-контейнером, что позволяет использовать его в веб-приложениях. Он предоставляет методы для взаимодействия с сервлет-контекстом (ServletContext).

- Поддержка ServletContext:
WebApplicationContext предоставляет доступ к ServletContext, что позволяет взаимодействовать с атрибутами сервлет-контекста, параметрами инициализации и другими сервлет-специфичными настройками.

- Поддержка конфигурации веб-приложений:
WebApplicationContext позволяет загружать конфигурацию из web.xml и других источников, специфичных для веб-приложений.

- Поддержка RootWebApplicationContext и ServletContext:
В веб-приложениях часто используется иерархия контекстов, где RootWebApplicationContext является корневым контекстом, а WebApplicationContext для конкретного сервлета может быть дочерним контекстом.


**Mapping**

1. @GetMapping - аннотация для сопоставления HTTP-запросов GET с определенными методами обработчика.
2. @PostMuping - аннотация для сопоставления HTTP-запросов POST с определенными методами обработчика.
3. @PutMapping - аннотация для сопоставления HTTP-запросов PUT с определенными методами обработчика.
4. @DeleteMapping - аннотация для сопоставления HTTP-запросов DELETE с конкретными методами обработчика.
5. @PatchMapping - аннотация для сопоставления HTTP-запросов PATCH с определенными методами обработчика.

**Основные параметры:**

- value и path:

Оба параметра используются для указания пути URL, который будет обрабатываться методом. value является основным параметром, но можно использовать path как синоним.

- params:

Этот параметр позволяет указать, какие параметры запроса должны присутствовать, чтобы метод был вызван.
Пример: @GetMapping(value = "/search", params = "q") — метод будет вызван только если параметр q присутствует в запросе.

- headers:

Позволяет указать, какие заголовки HTTP-запроса должны присутствовать для вызова метода.
Пример: @GetMapping(value = "/secure", headers = "Authorization")

- produces:

Указывает, какие типы содержимого (MIME-типы) метод может производить. Это полезно для указания типа данных, который будет возвращен клиенту.
Пример: @GetMapping(value = "/jsonData", produces = "application/json")

- consumes:

Указывает, какие типы содержимого метод может потреблять.
